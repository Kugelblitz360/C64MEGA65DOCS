C64 for MEGA65 User Documentation

Installation (INCOMPLETE)

  • Download the ZIP file that contains the bitstream and the core file and unpack it on your machine.
  • Choose the correct .cor file for your MEGA65 model: Please be aware that there are multiple MEGA65 models out there and that each .cor file only works with the model that it was built for. So in case you are not sure, what R3/R3A, R4 and R5 means and which model you have, head over to the model documentation to learn more.
  • Copy the .cor file on an SD card that has been formatted using the MEGA65’s built-in formatting tool. If you want to be on the safe side, just use the internal SD card (bottom tray), which is formatted like this by default.
  • Read the section “How do I install an alternative MEGA65 core?” on the alternative MEGA65 cores website or read the section “Bitstream Utility” in the MEGA65 Starter Guide.

Additional files

The Core support loading virtual disks, cartridges and single programs from both SD card slots if they are formated in FAT32. We recommend formatting either with the MEGA65 directly or using the SD Card Formatter tool from the SD Card Association. The formatting options of your operating system might not create a 100% compliant FAT32 filesystem.

https://www.sdcard.org/downloads/formatter/

After formatting please create a folder call c64 in the root directory of the SD Card. The Core will use this as the default/start folder for your C64 content.

Also please copy the config.file called c64mega65 from the ZIP file you downloaded into the c64 folder on the SD card. This is necessary because the Core can not create new files, it can only change files that already are on the card. If you do not copy this file, the Core can not save it’s settings and you will lose them between reboots.

Optionally you can add install JiffyDOS or a Real-Time-Clock driver for Geos RTC.

To configure the Core after installing it, just start it and press the HELP key anytime.

When changing SD cards

You can change SD Cards while running the Core, but this will have the side effect that your Config-file can not be saved to the new Card. The Config-file is only read once at the boot of the Core and later modified if you change options so changes might end on a different SD Card than the one you intend.

Config file

If you want the core to remember the settings you made in the on-screen-menu, then make sure that you copy the config file into a folder called /c64. This /c64 folder needs to be located in the root folder of the SD card that is active when you boot the core. The config file is called c64mega65 and is located in the ZIP file that you downloaded from the MEGA 65 filehost.

The very first time you start the core after you copied the c64mega65 config file into the /c64 folder, it will detect the new config file and present you with factory default settings when you press Help to open the on-screen-menu. After that, the core will always save the settings when you close the on-screen-menu and remember them when you restart the core at a later time.

Important: If you change the SD card, for example by using F1 or F3 in the file browser or by physically changing or removing the card that contains your /c64/c64mega65 config file, then the core will not remember your newest on-screen-menu settings upon next start.
